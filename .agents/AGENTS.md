# 全局 AI 助手规则

## ⚠️ 重要安全规则

### 代码修改前必须获得确认

**在没有获得我明确确认之前，不要修改任何代码文件。**

这包括但不限于：
- 不要执行 `WriteFile` 写入代码文件
- 不要执行 `StrReplaceFile` 修改代码文件
- 不要删除代码文件
- 不要重命名代码文件

### 允许的操作（无需确认）

以下操作可以在没有确认的情况下执行：
- ✅ 读取文件 (`ReadFile`)
- ✅ 搜索代码 (`Grep`)
- ✅ 列出文件 (`Glob`)
- ✅ 执行 Shell 命令查看信息（如 `git status`、`ls` 等）
- ✅ 网页搜索 (`SearchWeb`)
- ✅ 抓取网页 (`FetchURL`)

### 修改前的确认流程

当你需要修改代码时：

1. **先说明计划** - 告诉我你打算修改什么、为什么修改
2. **展示变更** - 使用 diff 格式或清晰描述展示将要进行的修改
3. **等待确认** - 等待我明确回复 "可以"、"确认"、"执行" 等
4. **执行修改** - 获得确认后再执行实际的文件修改

### 例外情况

只有在以下情况下可以跳过确认：
- 我明确使用了 `/yolo` 命令开启 YOLO 模式
- 我明确说了 "直接改"、"不需要确认" 等
- 创建新文件（非修改现有文件）

## ⚠️ Git 使用规则

### Git 上传命令需要确认

**在没有获得我明确允许之前，不要执行任何 Git 上传命令。**

需要确认的命令包括：
- `git push`
- `git commit`
- `git merge`
- `git rebase`
- `git reset`
- `git cherry-pick`
- 任何修改远程仓库状态的命令

### 允许的 Git 操作（无需确认）

以下 Git 查看操作可以在没有确认的情况下执行：
- ✅ `git status` - 查看状态
- ✅ `git log` - 查看日志
- ✅ `git branch` - 查看分支
- ✅ `git diff` - 查看差异
- ✅ `git remote -v` - 查看远程仓库
- ✅ `git stash list` - 查看 stash 列表

## 💬 回复风格规则

### 保持简洁
- 回复要简短明了，避免冗长
- 不要重复说明已经清楚的内容
- 避免不必要的客套话

### 代码示例必须带注释
- 所有代码示例必须包含中文注释
- 解释关键逻辑和用途
- 复杂代码添加行内注释

## 🖥️ 系统环境规则

### Windows 11 + PowerShell
- 本系统为 **Windows 11**
- 优先使用 **PowerShell** 命令
- 避免使用 bash/sh 命令
- Shell 命令使用 PowerShell 语法

---

## ✏️ 代码修改规则

### 最小修改原则
- 每次只修改必要的代码
- 不要修改与当前任务无关的文件
- 不要重构未涉及的功能
- 保持现有代码风格，不要批量格式化

---

## 编码规范

- 所有注释和文档字符串使用中文
- 变量名、函数名使用英文
